# 视频生成规则

## 命名规范

- 视频文件：`视频-{镜头名}-{版本}.mp4`
- 提示词文件：`视频-{镜头名}-{版本}.md`
- 存放目录：`<项目名>/4-视频/`

## 参考图规则

- 必须传入参考图（关键帧、故事板、或角色+场景参考）
- 参考图描述必须按 `--ref` 参数顺序，使用"图片1"、"图片2"等
- 多参考类型时，先列所有参考图，再列参考视频/音频

## Seedance 特定规则

- 使用 `@图片1`、`@视频1`、`@音频1` 等官方命名
- 必须在提示词中明确说明每个素材的用途
- 不要上传写实真人脸部素材
- 视频延长时，生成时长应为"新增部分"的时长

## 时长规则

| 总时长 | 推荐分段 |
|--------|----------|
| ≤15秒 | 单段生成 |
| 16-30秒 | 2段（首段15秒 + 延长段） |
| 31-45秒 | 3段 |
| 46-60秒 | 4段 |
| >60秒 | 拆分为独立场景分别生成 |

## 提示词规则

- 所有提示词必须使用中文
- 描述要具体且有画面感
- 镜头语言和动作描述要有时间顺序
- 台词/对白用引号包裹，并标注角色和情绪
- 音效描述单独成行
- **写完提示词文件后必须向用户确认**：在调用 API 之前，必须将写好的提示词文件展示给用户确认。只有在用户明确说出"生成""提交""生成吧""确认""OK"等指令后，才能调用 API。绝对禁止未经用户明确同意就主动提交 API 调用，避免浪费 token 和生成费用

## 基于故事板/分镜图生成视频的规则

当用户说"参考分镜图/故事板生成视频提示词"时，必须遵循以下规则：

- **保持分镜结构**：视频提示词的镜头设计部分必须与故事板的分镜一一对应。每个故事板镜头对应视频提示词中的一个独立镜头，不要合并为单镜头时间线。
- **每个镜头独立描述**：每个镜头有自己的画面描述、台词/画外音、景别、机位等信息，不要用 `0.0-X秒` 时间戳把所有内容塞进一个镜头。
- **单镜头场景例外**：只有当一个场景确实是单一连续镜头（如 ICU 主观视角长镜头）时，才使用单镜头 + 时间戳分段的格式。
- **分镜式格式示例**：
  ```
  ### 镜头1：纯黑开场
  **景别**：...
  **动作**：全黑画面...
  **台词/画外音**：...

  ### 镜头2：缓缓睁眼
  **景别**：...
  **动作**：...
  **台词/画外音**：...
  ```
