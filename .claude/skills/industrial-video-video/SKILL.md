# 视频生成工作流

基于关键帧或故事板生成最终视频。支持 Seedance 2.0 等视频生成模型。

## 输出物

- 视频文件（MP4）
- 视频提示词文件（.md）

## 适用场景

- 关键帧已确认，需要生成动态视频
- 故事板已确认，需要生成完整镜头
- 需要基于参考图生成特定动作的视频

---

## 核心规则（内联）

### 命名规范

| 类型 | 命名格式 | 示例 | 存放目录 |
|------|----------|------|----------|
| 视频文件 | `视频-{镜头名}-{版本}.mp4` | `视频-街头行走-v1.mp4` | `<项目名>/4-视频/` |
| 提示词文件 | 与视频同名，`.md` 后缀 | `视频-街头行走-v1.md` | 与输出文件同目录 |

### 项目级风格锁定

每个 `项目-*/` 目录下必须有 `项目说明.md`，其中必须包含 `## 画面风格提示词` 段落。

- 默认行为：生成任何内容时，自动将该风格追加到 prompt 末尾
- 除非用户**显式说**「不参考项目风格」或「不用项目风格」，否则必须注入
- 风格注入位置：prompt 最后，用「，」连接
- **项目风格提示词必须完整复制，不得简化、缩写或概括**

### 提示词文件结构（强制）

每个提示词文件必须包含以下两部分：

**1. YAML frontmatter（文件头）**：
```markdown
---
model: {模型ID}
duration: {秒数}
ratio: "{宽高比}"
resolution: {分辨率}
ref-images:
  - {角色参考图路径}
  - {场景参考图路径}
  - {首帧关键帧路径（可选）}
  - {尾帧关键帧路径（可选）}
ref-audios: []
ref-videos: []
seed: null
status: null
task_id: null
---
```

**2. Markdown 内容（分层结构）**：

Prompt 内容分散在各分层章节中，**不需要单独的"完整提示词"汇总章节**。各章节共同构成完整的生成指令。

```markdown
# 视频-{镜头名} Prompt

## 参考图

- **图片1**：{角色名} 人物参考
- **图片2**：{场景名} 场景参考
- **图片3**：（可选）首帧关键帧
- **图片4**：（可选）尾帧关键帧

## 视觉风格

{项目风格提示词}

## 人物外貌

{从角色参考图提取的外貌描述}

## 环境描述

{从场景参考图提取的环境描述}

## 镜头设计

### 镜头{N}：{镜头名}（{时长}秒）

**景别**：{特写/近景/中景/全景/远景}
**机位**：{固定/运动，相机高度、角度、与主体的距离}
**构图**：{主体在画面中的位置、占比}
**焦点**：{焦点位置，景深效果}
**动作**：
- {0.0-X秒：只描述画面中可见元素的动作}
- {X-Y秒：...}
**台词/画外音**：{角色对话、内心独白、旁白、字幕文字等。视频生成模型支持语音和字幕，如有台词必须在此明确列出，不得省略}
**光影**：{光源方向、强度、色温，阴影位置}

## 负面 Prompt

{需要排除的元素}
```

### API 调用确认规则（重要）

- **写完提示词文件后，必须先将提示词内容展示给用户确认**
- **只有在用户明确说出"生成""提交""生成吧""确认""OK"等指令后，才能调用 API**
- **绝对禁止未经用户明确同意就主动提交 API 调用，这会直接浪费 token 和生成费用**
- 如果用户只给了一句话描述，必须先要求补充结构化信息或向用户确认，禁止直接生成

### 参考图规则

- 必须传入参考图（关键帧、故事板、或角色+场景参考）
- 参考图描述必须按 `--ref` 参数顺序，使用"图片1"、"图片2"等，**绝不能出现文件名**
- 多参考类型时，先列所有参考图，再列参考视频/音频

### 图生图 / 图生视频 Prompt 规则

- 使用 `--ref` 提供参考图时，prompt **只描述需要修改/新增的内容**
- 不要复述参考图里已有的外观特征
- **输出路径和参考图路径必须不同**，禁止覆盖原图

### 参考图描述规范

```
参考图共2张：
图1：角色正面全身照...
图2：古风庭院场景...

请生成视频：图1中的角色站在图2的场景中...
```

### 时长规则

| 总时长 | 推荐分段 |
|--------|----------|
| ≤15秒 | 单段生成 |
| 16-30秒 | 2段（首段15秒 + 延长段） |
| 31-45秒 | 3段 |
| 46-60秒 | 4段 |
| >60秒 | 拆分为独立场景分别生成 |

### 禁止一句话 Prompt

任何生成请求，如果用户只给了一句话描述，必须**拒绝直接生成**，而是要求用户补充结构化信息，或基于上下文推断后**向用户确认**再执行。

## 生成流程

### Step 1 — 确认视频参数

| 参数 | 必问 | 选项 |
|------|------|------|
| 时长 | ✅ | 4-8秒 / 9-12秒 / 13-15秒 / 超长（分段） |
| 比例 | ✅ | 16:9 / 9:16 / 1:1 |
| 参考素材 | ✅ | 关键帧 / 故事板 / 角色+场景参考 |
| 风格 | 可选 | 情绪氛围、镜头风格 |

### Step 2 — 编写提示词文件

按 [prompt-template.md](prompt-template.md) 模板或 [seedance.md](seedance.md) 编写。

### Step 3 — 生成视频

```bash
npx -y bun .claude/skills/content-gen/scripts/video/main.ts \
  --promptfile 视频-{镜头名}-{版本}.md \
  --video 项目-xxx/4-视频/视频-{镜头名}-{版本}.mp4 \
  --ref 项目-xxx/3-关键帧/关键帧-{镜头名}-v1.png \
  --ar 16:9 --duration 5
```

## Seedance 特定规则

- 使用 `@图片1`、`@视频1`、`@音频1` 等官方命名
- 必须在提示词中明确说明每个素材的用途
- 不要上传写实真人脸部素材
- 视频延长时，生成时长应为"新增部分"的时长
- 详细规则见 [seedance.md](seedance.md)

## 质量检查清单

- [ ] 视频时长符合要求

- [ ] 场景风格与参考图一致
- [ ] 动作连贯自然
- [ ] 画面风格与项目风格一致
