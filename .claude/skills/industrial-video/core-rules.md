# 核心规则

## 1. 任务类型判断（第一步，必须先执行）

收到用户任何生成指令时，**首先判断任务类型**，再决定后续动作。

| 任务类型 | 判断依据 | 所需参考 |
|----------|----------|----------|
| **文生图** | 用户要求生成全新图片，无参考图 | 项目风格（可选） |
| **图生图** | 用户要求基于现有图片修改/变体 | **必须传入参考图** |
| **图生视频** | 用户要求基于图片生成视频 | **必须传入参考图** |
| **文生视频** | 用户要求纯文字生成视频（极少用） | 项目风格（必须） |

**关键规则**：
- 如果任务涉及「基于某个场景/角色生成」，那就是**图生图或图生视频**，必须传入参考图
- 如果用户说「参考第X格」「像这个风格」「在这个场景里」，必须传入对应参考图
- **禁止在需要参考图的任务中只用文字描述**

## 2. 禁止一句话 Prompt（铁律）

任何生成请求，如果用户只给了一句话描述，Agent 必须**拒绝直接生成**，而是要求用户补充结构化信息，或基于上下文推断后**向用户确认**再执行。

❌ 错误示例：`生成一只猫在花园里的视频`

✅ 正确流程：
> Agent：请补充以下信息：
> - 猫的外貌（品种、颜色、体型、眼睛颜色）
> - 服装/配饰（如有）
> - 花园场景（风格、季节、光照、天气）
> - 猫的动作/表情
> - 画面风格偏好

## 3. 项目级风格锁定

每个 `项目-*/` 目录下必须有 `项目说明.md`，其中必须包含 `## 画面风格提示词` 段落。

**规则**：
- 默认行为：生成任何内容时，自动将该风格追加到 prompt 末尾
- 除非用户**显式说**「不参考项目风格」或「不用项目风格」，否则必须注入
- 风格注入位置：prompt 最后，用「，」连接
- **项目风格提示词必须完整复制，不得简化、缩写或概括**。即使提示词很长，也必须原样完整注入

**示例**：
- 项目风格：`2D 动画风格，细腻手绘质感，吉卜力色调`
- 用户 prompt：`一只橘猫在樱花树下`
- 实际提交：`一只橘猫在樱花树下，2D 动画风格，细腻手绘质感，吉卜力色调`

## 4. 提示词分层结构（强制）

所有提示词文件必须按以下分层结构编写：

```markdown
## 参考图

- **图片1**：{描述} `{绝对路径}`
- **图片2**：{描述} `{绝对路径}`
（如有多个参考图，按图片1/图片2/...编号）

## 视觉风格

{画面整体的艺术风格、渲染方式、绘画技法}

## 人物外貌

{人物的长相、发型、五官、表情、穿着、配饰}

## 环境描述

{场景、背景、地点、时间、氛围}

## 人物动作

{姿态、动作、肢体语言、与镜头的互动关系}

## 光影构图

{光源、明暗、镜头角度、景别、画面布局}

## 完整提示词

{将以上各层合并为一段完整的中文提示词，按重要性排序，用逗号分隔}

## 负面 Prompt

{需要排除的元素}
```

**分层规则**：
- 提示词必须为纯中文
- 每层一段描述性文字，不要分点罗列（完整提示词层除外）
- 按重要性排序：视觉风格 > 人物外貌 > 人物动作 > 环境描述 > 光影构图 > 色彩 > 细节
- 如果某层不适用当前任务（如场景生成没有人物），写"依具体场景而定"或省略
- 参考图命名必须按 `--ref` 参数中的提交顺序，使用"图片1"、"图片2"等，**绝不能出现文件名**

## 5. 命名规范（强制）

所有生成的文件必须按以下规则命名：

| 类型 | 命名格式 | 示例 | 存放目录 |
|------|----------|------|----------|
| 人物参考图 | `人物参考-{人物名}-{特征}.png` | `人物参考-Yi-运动服.png` | `<项目名>/2-素材/角色/{人物名}/` |
| 场景参考图 | `场景参考-{场景名}-{视角}.png` | `场景参考-别墅大门-正面视角.png` | `<项目名>/2-素材/场景/{场景名}/` |
| 关键帧 | `关键帧-{镜头名}-{版本}.png` | `关键帧-街头行走-v1.png` | `<项目名>/3-关键帧/` |
| 视频 | `视频-{镜头名}-{版本}.mp4` | `视频-街头行走-v1.mp4` | `<项目名>/4-视频/` |
| 故事板 | `故事板-{镜头名}-{版本}.png` | `故事板-街头行走-v1.png` | `<项目名>/3-关键帧/` |
| 提示词文件 | 与对应图片/视频同名，`.md` 后缀 | `人物参考-Yi-运动服.md` | 与输出文件同目录 |

**规则**：
- 除了**自由生图**之外，其他任务一律先生成提示词的 markdown 文件
- 提示词文件命名与对应图片/视频相同，仅后缀不同（`.md` vs `.png`/`.mp4`）
- 版本号从 v1 开始，每次迭代递增

## 6. 风格一致性控制清单

生成任何内容前，Agent 必须检查：

- [ ] 当前是否在 `项目-*/` 目录下？
- [ ] `项目说明.md` 是否存在？
- [ ] `## 画面风格提示词` 是否非空？
- [ ] 用户是否显式说「不参考项目风格」？
- [ ] Prompt 是否覆盖视觉风格/人物外貌/人物动作三个维度（如适用）？
- [ ] 参考图是否已确认并传入？
- [ ] Prompt 文件是否已写入？
- [ ] 文件命名是否符合命名规范？
- [ ] 生图/生视频的分辨率参数是否已确认（默认 2k / 1080p）？

## 7. Agent 自我约束：空间与物理能力边界

**物理直觉和空间想象能力非常弱**。

凡是涉及以下领域的工作，**不要亲自推断、计算或可视化**，而是把需求写入 prompt，让生图/生视频 AI 自行处理：

- 三维空间关系（物体相对位置、深度、透视）
- 物理运动（惯性、加速度、碰撞、反弹轨迹）
- 人体姿态/肢体角度（手脚具体摆放位置、关节弯曲角度）
- 光影的物理传播（阴影方向、反射折射路径、丁达尔效应的精确形态）
- 任何需要"脑内渲染"才能确认的画面效果

**正确做法**：用文字描述需求（景别、机位、构图、动作方向、光影氛围），让 AI 模型自行完成空间排列和物理合理性判断。不要在 prompt 中硬写具体像素坐标或精确几何参数。
