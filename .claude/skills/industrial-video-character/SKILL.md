# 角色生成工作流

为视频生成提供角色外观一致性控制。包含三视图（正面/侧面/背面）和多种表情/姿态的完整角色视觉资料，确保 AI 视频生成时角色不会变形或走样。

## 输出物

- 人物参考图大图（PNG）
- 人物提示词文件（.md）
- 可选：切割后的独立图片

## 适用场景

- 新角色首次出场，需要建立视觉基准
- 角色外观需要多角度展示
- 角色需要多种表情和姿态参考
- 作为视频生成的 `--ref-image` 参考图输入

## 核心规则（内联）

### 命名规范

| 类型 | 命名格式 | 示例 | 存放目录 |
|------|----------|------|----------|
| 人物参考图 | `人物参考-{人物名}-{特征}.png` | `人物参考-Yi-运动服.png` | `<项目名>/2-素材/角色/{人物名}/` |
| 提示词文件 | 与图片同名，`.md` 后缀 | `人物参考-Yi-运动服.md` | 与输出文件同目录 |

### 项目级风格锁定

每个 `项目-*/` 目录下必须有 `项目说明.md`，其中必须包含 `## 画面风格提示词` 段落。

- 默认行为：生成任何内容时，自动将该风格追加到 prompt 末尾
- 除非用户**显式说**「不参考项目风格」或「不用项目风格」，否则必须注入
- 风格注入位置：prompt 最后，用「，」连接
- **项目风格提示词必须完整复制，不得简化、缩写或概括**

### 提示词分层结构（强制）

所有提示词文件必须按以下分层结构编写：

```markdown
## 参考图

（文生图，无参考图）

## 视觉风格

{画面整体的艺术风格、渲染方式、绘画技法}

## 人物外貌

{人物的长相、发型、五官、表情、穿着、配饰}

## 服装

{上衣、下装、配饰、鞋}

## 人物动作

{姿态、动作、肢体语言、与镜头的互动关系}

## 环境描述

{场景、背景、地点、时间、氛围}

## 光影构图

{光源、明暗、镜头角度、景别、画面布局}

## 完整提示词

{将以上各层合并为一段完整的中文提示词，按重要性排序，用逗号分隔}

## 负面 Prompt

{需要排除的元素}
```

**分层规则**：
- 提示词必须为纯中文
- 每层一段描述性文字，不要分点罗列（完整提示词层除外）
- 按重要性排序：视觉风格 > 人物外貌 > 人物动作 > 环境描述 > 光影构图 > 色彩 > 细节
- 参考图命名必须按 `--ref` 参数中的提交顺序，使用"图片1"、"图片2"等，**绝不能出现文件名**

### 禁止一句话 Prompt

任何生成请求，如果用户只给了一句话描述，必须**拒绝直接生成**，而是要求用户补充结构化信息，或基于上下文推断后**向用户确认**再执行。

## 布局类型

统一布局：左侧三视图 + 右侧 2×2 表情特写

```
┌──────────────────┬─────────────┬─────────────┐
│                  │   表情1      │   表情2      │
│   正面全身大图    ├─────────────┼─────────────┤
│   (1/2 宽度)      │   表情3      │   表情4      │
│                  │             │             │
│  包含：正面/侧面   │  (2×2 网格)  │             │
│      /背面小图    │  面部特写    │             │
└──────────────────┴─────────────┴─────────────┘
```

## 生成流程

### Step 1 — 确认角色描述

检查项目内是否已有 `描述.md`。如果没有，先创建：

```markdown
# 角色名

## 外貌
- 性别：
- 年龄：
- 发型：
- 五官：
- 体型：
- 特征：

## 服装
- 上衣：
- 下装：
- 配饰：
- 鞋：

## 性格/气质

## 常用表情
```

### Step 2 — 确认表情类型

右侧 2×2 网格需指定 4 种表情，常见组合：

| 位置 | 推荐表情 |
|------|---------|
| 左上 | 开心 / 微笑 |
| 右上 | 愤怒 / 冷峻 |
| 左下 | 悲伤 / 忧郁 |
| 右下 | 惊讶 / 冷漠 |

用户可自定义，需在提示词中明确列出。

### Step 3 — 编写提示词文件

按 [prompt-template.md](prompt-template.md) 模板编写。

### Step 4 — 生成图片

```bash
npx -y bun .claude/skills/content-gen/scripts/image/main.ts \
  --promptfile 人物参考-{角色名}-{特征}.md \
  --image 项目-xxx/2-素材/角色/{角色名}/人物参考-{角色名}-{特征}.png \
  --ar 1:1 --quality 2k
```

### Step 5 — 切割（可选）

```bash
npx -y bun .claude/skills/content-gen/scripts/image/grid.ts panels \
  人物参考-{角色名}-{特征}.png ./拆分 \
  --panels '[{"name":"正面","left":0,"top":0,"width":0.5,"height":0.7},{"name":"侧面","left":0,"top":0.7,"width":0.25,"height":0.3},{"name":"背面","left":0.25,"top":0.7,"width":0.25,"height":0.3},{"name":"表情1","left":0.5,"top":0,"width":0.25,"height":0.5},{"name":"表情2","left":0.75,"top":0,"width":0.25,"height":0.5},{"name":"表情3","left":0.5,"top":0.5,"width":0.25,"height":0.5},{"name":"表情4","left":0.75,"top":0.5,"width":0.25,"height":0.5}]'
```

## 文件存放

```
项目-漫剧-xxx/2-素材/角色/<角色名>/
├── 参考图.png           ← 人物参考图大图（必须）
├── 描述.md              ← 结构化文字描述（必须）
└── 拆分/                ← 切割后的独立图片（可选）
    ├── 正面.png
    ├── 侧面.png
    ├── 背面.png
    ├── 开心.png
    └── ...
```

## 质量检查清单

- [ ] 左侧三视图（正面/侧面/背面）完整可见
- [ ] 右侧 4 种表情特写清晰可辨，突出面部五官细节
- [ ] 所有面板中角色服装一致
- [ ] 仅表情和姿态变化，外貌不变
- [ ] 背景统一为纯色（白/灰/透明）
- [ ] 画面风格与项目风格一致
