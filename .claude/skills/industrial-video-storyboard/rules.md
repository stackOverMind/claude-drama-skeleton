# 故事板生成规则

## 命名规范

- 故事板图：`故事板-{镜头名}-{版本}.png`
- 提示词文件：`故事板-{镜头名}-{版本}.md`
- 存放目录：`<项目名>/3-关键帧/`

## 输入规则

- 必须基于已有的关键帧或参考图生成
- 故事板生成单图模式：传入多个关键帧作为 `--ref`
- 故事板生成视频模式：传入关键帧序列或故事板图

## 分辨率规则

- **默认分辨率：normal**。除非用户明确要求高分辨率（如"高清""2K""高质量"），否则一律使用 `--quality normal`
- 故事板作为导演参考图，normal 质量已足够，无需追求过高分辨率导致生成超时或费用增加

## 一致性规则

- 所有关键帧中的角色外观必须一致
- 所有关键帧中的场景风格必须一致
- 镜头序列逻辑清晰，过渡自然

## 提示词规则

- 必须描述每个输入关键帧的内容
- 必须说明镜头序列的顺序和过渡方式
- 必须包含项目风格提示词
- **不指定排版格式**：不规定行列数、格子数量、排列方向（如"九宫格""从左到右""2行3列"等），只提供镜头内容和顺序，由生图AI自行决定最佳排版
- **写完提示词后必须向用户确认**：在调用 API 之前，必须将写好的提示词文件展示给用户确认。未经用户明确同意（如"生成吧""确认""OK"等），不得直接调用 API，避免浪费 token 和生成费用
