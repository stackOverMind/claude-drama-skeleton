# 故事板生成工作流

基于关键帧或参考图生成故事板，用于视频生成或导演参考。故事板可以是单张组合图，也可以是基于关键帧序列生成的视频。

## 输出物

- 故事板图片（PNG）或视频（MP4）
- 故事板提示词文件（.md）

## 适用场景

- 需要将多个关键帧组合为一张导演参考图
- 基于关键帧序列生成连贯视频
- 预览镜头之间的过渡效果

---

## 核心规则（内联）

### 命名规范

| 类型 | 命名格式 | 示例 | 存放目录 |
|------|----------|------|----------|
| 故事板图 | `故事板-{镜头名}-{版本}.png` | `故事板-街头行走-v1.png` | `<项目名>/3-关键帧/` |
| 提示词文件 | 与图片同名，`.md` 后缀 | `故事板-街头行走-v1.md` | 与输出文件同目录 |

### 项目级风格锁定

每个 `项目-*/` 目录下必须有 `项目说明.md`，其中必须包含 `## 画面风格提示词` 段落。

- 默认行为：生成任何内容时，自动将该风格追加到 prompt 末尾
- 除非用户**显式说**「不参考项目风格」或「不用项目风格」，否则必须注入
- 风格注入位置：prompt 最后，用「，」连接
- **项目风格提示词必须完整复制，不得简化、缩写或概括**

### 提示词文件结构（强制）

每个提示词文件必须包含以下两部分：

**1. YAML frontmatter（文件头）**：
```markdown
---
model: {模型ID}
ratio: "{宽高比}"
resolution: {分辨率}
ref-images:
  - {参考图路径1}
  - {参考图路径2}
seed: null
status: submitted
task_id: null
---
```

**2. Markdown 内容（分层结构）**：

```markdown
# 故事板-{镜头名} Prompt

## 参考图

- **图片1**：关键帧1（{描述}）
- **图片2**：关键帧2（{描述}）

## 视觉风格

{项目风格提示词}

## 人物外貌

{从角色参考图提取的外貌描述}

## 环境描述

{从场景参考图提取的环境描述}

## 故事板画面

{镜头序列描述}

## 负面 Prompt

{需要排除的元素}
```

### 参考图规则

- 必须基于已有的关键帧或参考图生成
- 故事板生成单图模式：传入多个关键帧作为 `--ref`
- 故事板生成视频模式：传入关键帧序列或故事板图
- prompt 中按 `--ref` 参数顺序，使用"图片1"、"图片2"等，**绝不能出现文件名**

### 禁止一句话 Prompt 与 API 调用确认

任何生成请求，如果用户只给了一句话描述，必须**拒绝直接生成**，而是要求用户补充结构化信息，或基于上下文推断后**向用户确认**再执行。

**重要：写完提示词文件后，必须先将提示词内容展示给用户确认。只有在用户明确说出"生成""提交""生成吧""确认""OK"等指令后，才能调用 API。绝对禁止未经用户明确同意就主动提交 API 调用，这会直接浪费 token 和生成费用。**

## 两种模式

### 模式一：故事板生成单图

将多个关键帧或镜头组合为一张故事板大图。

```bash
npx -y bun .claude/skills/content-gen/scripts/image/main.ts \
  --promptfile 故事板-{镜头名}.md \
  --image 项目-xxx/3-关键帧/故事板-{镜头名}.png \
  --ref 关键帧1.png 关键帧2.png ... \
  --ar 16:9 --quality normal
```

### 模式二：故事板生成视频

基于故事板图或关键帧序列生成视频。

```bash
# 先生成故事板图，再生成视频
# 或直接用关键帧序列生成视频
```

## 生成流程

### Step 1 — 确认故事板内容

```markdown
# 故事板

## 镜头序列
1. [画面描述]
2. [画面描述]
3. ...

## 参考图
- 关键帧1：[路径]
- 关键帧2：[路径]
- ...
```

### Step 2 — 编写提示词文件

按 [prompt-template.md](prompt-template.md) 模板编写。

### Step 3 — 生成

根据模式选择生成图片或视频。

---

## 一致性规则

- 所有关键帧中的角色外观必须一致
- 所有关键帧中的场景风格必须一致
- 镜头序列逻辑清晰，过渡自然
- **不指定排版格式**：不规定行列数、格子数量、排列方向（如"九宫格""从左到右""2行3列"等），只提供镜头内容和顺序，由生图AI自行决定最佳排版
