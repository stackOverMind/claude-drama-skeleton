# 场景生成工作流

为视频生成提供场景视觉一致性控制。围绕同一地点从不同角度、方位、景别建立完整视觉资料，确保 AI 视频生成时场景不会变形或风格漂移。

## 输出物

- 场景参考图大图（PNG）
- 场景提示词文件（.md）
- 可选：切割后的独立图片

## 适用场景

- 新场景首次出现，需要建立视觉基准
- 场景需要多角度参考以确保空间一致性
- 场景包含重要细节需要多角度展示
- 作为视频生成的 `--ref-image` 参考图输入

---

## 核心规则（内联）

### 命名规范

| 类型 | 命名格式 | 示例 | 存放目录 |
|------|----------|------|----------|
| 场景参考图 | `场景参考-{场景名}-{视角}.png` | `场景参考-别墅大门-正面视角.png` | `<项目名>/2-素材/场景/{场景名}/` |
| 提示词文件 | 与图片同名，`.md` 后缀 | `场景参考-别墅大门-正面视角.md` | 与输出文件同目录 |

### 项目级风格锁定

每个 `项目-*/` 目录下必须有 `项目说明.md`，其中必须包含 `## 画面风格提示词` 段落。

- 默认行为：生成任何内容时，自动将该风格追加到 prompt 末尾
- 除非用户**显式说**「不参考项目风格」或「不用项目风格」，否则必须注入
- 风格注入位置：prompt 最后，用「，」连接
- **项目风格提示词必须完整复制，不得简化、缩写或概括**

### 提示词分层结构（强制）

所有提示词文件必须按以下分层结构编写：

```markdown
## 参考图

（文生图，无参考图）

## 视觉风格

{画面整体的艺术风格、渲染方式、绘画技法}

## 人物外貌

依具体场景而定（场景图中无人物或人物很小）

## 环境描述

{场景类型、地点、建筑/地形、植被、天气、时间}

## 人物动作

无

## 光影构图

{光源方向、时间、天气对应的光照效果}

## 完整提示词

{将以上各层合并为一段完整的中文提示词}

## 负面 Prompt

{需要排除的元素}
```

**分层规则**：
- 提示词必须为纯中文
- 每层一段描述性文字，不要分点罗列（完整提示词层除外）
- 按重要性排序：视觉风格 > 环境描述 > 光影构图 > 色彩 > 细节

### 禁止一句话 Prompt

任何生成请求，如果用户只给了一句话描述，必须**拒绝直接生成**，而是要求用户补充结构化信息，或基于上下文推断后**向用户确认**再执行。

## 场景分类

场景不再区分「主/次」，只按空间类型分为两类：

- **室内场景**：封闭/半封闭空间，如房间、走廊、洞穴、车厢、狭长的街道等
- **室外场景**：开放空间，如广场、原野、海滩、城市天际线等

> 狭长街道视为室内场景——两侧建筑形成围合感，需要俯视平面图来确立空间关系。

---

## 布局类型

### 室内场景 — 单张参考板（左侧俯视 + 右侧多机位）

室内空间只生成**一张参考板大图**，左侧为俯视全景确立空间布局，右侧为 2×2 多机位视角，确保不同角度下的空间一致性。

```
┌─────────────────┬─────────────┬─────────────┐
│                 │   机位A全景   │   机位B侧面  │
│    俯视全景图    ├─────────────┼─────────────┤
│  （空间结构+家具  │   机位C近景   │   机位D特写  │
│     位置布局）   │             │             │
└─────────────────┴─────────────┴─────────────┘
        ↑ 占2列高度              ↑ 右侧 2×2 四宫格
```

**机位设计原则**：
- 所有机位必须基于左侧俯视图的空间关系
- 门窗、家具、装饰物在不同机位中的相对位置必须一致
- 机位角度应有明显区分（正面/侧面/斜角/特写），避免重复

> 只生成**一张**参考板，不要拆分成多张。

---

### 室外场景 — 多角度空间参考（3×3）

开放空间同样以**空间一致性**为核心，围绕同一地点从不同方位、景别建立视觉基准：

```
┌─────────────┬─────────────┬─────────────┐
│   全景远景   │   中景构图   │   近景细节   │  ← 第1行：景别
├─────────────┼─────────────┼─────────────┤
│   正面角度   │   侧面角度   │   斜角构图   │  ← 第2行：方位角度
├─────────────┼─────────────┼─────────────┤
│   低机位仰拍  │   平视机位   │   高机位俯拍  │  ← 第3行：机位高度
└─────────────┴─────────────┴─────────────┘
```

---

### 简化版 — 四宫格（2×2）

当场景出现频次低、或需要快速生成时，使用简化版：

```
┌─────────────┬─────────────┐
│   全景/正面   │   全景/侧面   │
├─────────────┼─────────────┤
│   中景/细节   │   近景/特写   │
└─────────────┴─────────────┘
```

## 生成流程

### Step 1 — 确认场景描述

创建或检查场景描述：

```markdown
# 场景名

## 地点
- 类型：（室内 / 室外）
- 规模：（房间/走廊/街道/建筑/广场/原野等）
- 地理位置：（如有特殊要求）

## 空间结构（室内场景必填）
- 房间形状：
- 门窗位置：
- 家具/陈设布局：
- 摄像机机位规划：

## 环境元素
- 建筑/地形：
- 植被：
- 天气/气候：
- 时间：

## 氛围
- 色调：
- 光线：
- 情绪：

## 关键细节
- 标志性元素：
- 道具：
```

### Step 2 — 选择布局

| 场景类型 | 输出物 | 布局 | 理由 |
|---------|--------|------|------|
| **室内场景** | 1张参考板 | 左侧俯视全景 + 右侧 2×2 多机位 | 单张图内同时确立空间布局和多角度参考，确保空间一致性 |
| **室外场景** | 1张参考图 | 3×3 九宫格 或 2×2 四宫格 | 围绕同一地点从不同方位、景别、机位高度建立视觉基准，确保空间一致性 |

**注意**：狭长街道等两侧有围合感的空间，按室内场景处理——参考板左侧为俯视确立街道走向和建筑位置。

### Step 3 — 编写提示词文件

按 [prompt-template.md](prompt-template.md) 模板编写。

### Step 4 — 生成图片

**⚠️ 调用 API 前必须向用户确认**：
- 提示词文件内容是否正确
- `--ar` 是否与项目视频比例一致
- `--quality` 是否符合场景复杂度
- 输出路径是否正确
- **未经用户确认，不得直接调用 API**

> API 调用按尺寸计费，无论 `normal`/`2k`/`3k` 均有成本，确认前务必提醒用户。

**`--quality` 选择惯例**：
- `normal`（1024 base）— 单张简单参考图
- `2k`（2048 base）— 多宫格、故事板、人物参考图、场景参考图等复杂构图

**`--ar` 根据项目说明中的视频比例确定**。

**建议后台运行**：单张图生成耗时较长，使用 `run_in_background` 避免阻塞，完成后自动通知。

```bash
# 室内场景 — 参考板（左侧俯视 + 右侧多机位）
npx -y bun .claude/skills/content-gen/scripts/image/main.ts \
  --promptfile 场景参考-{场景名}-室内参考板.md \
  --image 项目-xxx/2-素材/场景/{场景名}/场景参考-{场景名}-室内参考板.png \
  --ar <项目视频比例> --quality 2k

# 室外场景 — 多角度参考（九宫格/四宫格）
npx -y bun .claude/skills/content-gen/scripts/image/main.ts \
  --promptfile 场景参考-{场景名}-{视角}.md \
  --image 项目-xxx/2-素材/场景/{场景名}/场景参考-{场景名}-{视角}.png \
  --ar <项目视频比例> --quality 2k

### Step 5 — 切割（可选）

```bash
# 九宫格切割
npx -y bun .claude/skills/content-gen/scripts/image/grid.ts split \
  场景参考-{场景名}-{视角}.png ./拆分 --grid 3x3 --prefix <场景名>

# 四宫格切割（室外简化版）
npx -y bun .claude/skills/content-gen/scripts/image/grid.ts split \
  场景参考-{场景名}-{视角}.png ./拆分 --grid 2x2 --prefix <场景名>
```

## 文件存放

**必须按项目实际目录结构存放，不要机械套用模板路径。**

生成前先检查项目内已有目录：
- 如果已有 `2-素材/场景/`，就放在 `2-素材/场景/<场景名>/`
- 如果已有 `3-素材/场景/`，就放在 `3-素材/场景/<场景名>/`
- 如果两者都不存在，按项目类型标准创建：
  - 单集视频 → `2-素材/场景/<场景名>/`
  - AI 漫剧 → `3-素材/场景/<场景名>/`

```
2-素材/场景/<场景名>/
├── 场景参考-{场景名}-室内参考板.png   ← 室内：左侧俯视+右侧多机位（必须）
├── 场景参考-{场景名}-{视角}.png      ← 室外：九宫格/四宫格大图（必须）
├── 描述.md                            ← 结构化文字描述（必须）
└── 拆分/                              ← 切割后的独立图片（可选）
    ├── <场景名>_cell_1_1.png
    ├── <场景名>_cell_1_2.png
    └── ...
```

## 质量检查清单

- [ ] 所有格子/机位展示的是同一地点
- [ ] 建筑/环境元素在不同角度下保持一致（空间一致性）
- [ ] 室内场景：俯视图与多机位图的空间关系吻合（门窗、家具位置一致）
- [ ] 室外场景：同一地标在不同方位角度下可识别
- [ ] 画面风格与项目风格一致
- [ ] 标志性细节在所有格子/机位中可识别
