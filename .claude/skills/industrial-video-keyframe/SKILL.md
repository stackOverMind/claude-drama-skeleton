# 关键帧生成工作流

将剧本中的单个镜头转化为关键帧画面，用于：
- 视频生成时的 `--ref-image` 或 `--ref-video` 输入
- 镜头之间的连贯性控制
- 导演意图的可视化传达

## 输出物

- 单张或多张关键帧图片（PNG）
- 关键帧描述文件（关键帧.md）
- 可选：按镜头切割的独立图片

## 适用场景

- 剧本已完成，需要将文字转化为视觉参考
- 复杂镜头需要精确控制画面构图
- 多镜头连续场景需要保持连贯性
- 作为 Seedance 等视频模型的参考输入

## 核心规则（内联）

### 命名规范

| 类型 | 命名格式 | 示例 | 存放目录 |
|------|----------|------|----------|
| 关键帧图 | `关键帧-{镜头名}-{版本}.png` | `关键帧-街头行走-v1.png` | `<项目名>/3-关键帧/` |
| 提示词文件 | 与图片同名，`.md` 后缀 | `关键帧-街头行走-v1.md` | 与输出文件同目录 |

### 项目级风格锁定

每个 `项目-*/` 目录下必须有 `项目说明.md`，其中必须包含 `## 画面风格提示词` 段落。

- 默认行为：生成任何内容时，自动将该风格追加到 prompt 末尾
- 除非用户**显式说**「不参考项目风格」或「不用项目风格」，否则必须注入
- 风格注入位置：prompt 最后，用「，」连接
- **项目风格提示词必须完整复制，不得简化、缩写或概括**

### 提示词分层结构（强制）

```markdown
## 参考图

- **图片1**：{角色名} 人物参考（{特征描述}）
- **图片2**：{场景名} 场景参考（{视角/时间}）

## 视觉风格

{项目风格提示词}

## 人物外貌

{从角色参考图提取的外貌描述}

## 环境描述

{从场景参考图提取的环境描述}

## 人物动作

{具体动作描述}

## 光影构图

{镜头角度、景别、光源、构图}

## 完整提示词

{将以上各层合并}

## 负面 Prompt

{需要排除的元素}
```

### 参考图规则

- 基于角色生成新姿势 → 必须传入角色参考图
- 基于场景生成角色动作 → 必须传入场景参考图
- 角色在场景中做动作 → 角色参考图 + 场景参考图都必须传入
- 任何涉及已有素材的生成任务，必须传入参考图
- **禁止在需要参考图的任务中只用文字描述**

### 图生图 Prompt 规则

- 使用 `--ref` 提供参考图时，prompt **只描述需要修改/新增的内容**
- 不要复述参考图里已有的外观特征（颜色、材质、风格、姿势、构图等）
- 模型会自己从参考图里提取这些信息，重复描述反而会干扰生成
- **输出路径和参考图路径必须不同**，禁止覆盖原图

### 参考图描述规范

当使用单张九宫格作为参考时，prompt 中必须逐格描述：

```
参考图是一张3x3九宫格，从上往下、从左往右依次是：
图1：角色正面站姿...
图2：角色侧面站姿...
...
```

当使用多张独立图片时：

```
参考图共2张：
图1：角色正面全身照...
图2：古风庭院场景...
```

### 禁止一句话 Prompt

任何生成请求，如果用户只给了一句话描述，必须**拒绝直接生成**，而是要求用户补充结构化信息，或基于上下文推断后**向用户确认**再执行。

## 布局类型

### 单镜头单帧

每个镜头生成一张关键帧，最精确的控制方式。

```
镜头1: [画面描述] → 生成 镜头1_关键帧.png
镜头2: [画面描述] → 生成 镜头2_关键帧.png
镜头3: [画面描述] → 生成 镜头3_关键帧.png
```

### 镜头序列网格

将同一场景的多个镜头放在一张图上，便于整体审视连贯性。

```
┌─────────────┬─────────────┬─────────────┐
│   镜头1      │   镜头2      │   镜头3      │  ← 同一场景的分镜序列
│  开场全景    │  主角中景    │  表情特写    │
└─────────────┴─────────────┴─────────────┘
```

### 首尾帧对照

用于需要精确控制开始和结束画面的镜头。

```
┌─────────────────┬─────────────────┐
│     首帧         │     尾帧         │
│   (起始画面)      │   (结束画面)      │
└─────────────────┴─────────────────┘
```

### 动作分解关键帧

复杂动作镜头需要分解展示，用箭头标注四肢运动方向。

```
┌─────────────┬─────────────┬─────────────┬─────────────┐
│   动作1      │   动作2      │   动作3      │   动作4      │
│  起始姿势    │  中间姿势    │  中间姿势    │  结束姿势    │
│   ↗ 手臂    │   → 腿部    │   ↘ 躯干    │   ✓ 完成    │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

## 生成流程

### Step 1 — 从剧本提取镜头信息

每个镜头需要以下信息：

```markdown
# 镜头 [编号]

## 画面内容
- 场景：[场景名]
- 角色：[出场角色]
- 动作：[角色动作]
- 对话：[如有]

## 镜头语言
- 景别：（远景/全景/中景/近景/特写）
- 角度：（平视/俯视/仰视/侧视）
- 运动：（固定/推/拉/摇/移/跟）
- 时长：[秒数]

## 参考图
- 角色参考：[路径]
- 场景参考：[路径]
```

### Step 2 — 编写提示词文件

按 [prompt-template.md](prompt-template.md) 模板编写。

### Step 3 — 生成图片

```bash
# 单镜头
npx -y bun .claude/skills/content-gen/scripts/image/main.ts \
  --promptfile 关键帧-{镜头名}-{版本}.md \
  --image 项目-xxx/3-关键帧/关键帧-{镜头名}-{版本}.png \
  --ar 16:9 --quality 2k

# 使用角色/场景参考图
npx -y bun .claude/skills/content-gen/scripts/image/main.ts \
  --promptfile 关键帧-{镜头名}-{版本}.md \
  --image 项目-xxx/3-关键帧/关键帧-{镜头名}-{版本}.png \
  --ref 项目-xxx/2-素材/角色/<角色名>/人物参考-{角色名}-{特征}.png \
  --ar 16:9 --quality 2k
```

### Step 4 — 切割（如使用网格布局）

```bash
# 镜头序列网格切割
npx -y bun .claude/skills/content-gen/scripts/image/grid.ts split \
  关键帧序列.png ./拆分 --grid 3x3 --prefix 镜头

# 动作分解切割（4 个步骤）
npx -y bun .claude/skills/content-gen/scripts/image/grid.ts split \
  动作分解.png ./拆分 --grid 2x2 --prefix 动作
```

## 关键帧描述文件格式

```markdown
# 关键帧汇总

## 场景：[场景名]

### 镜头1
- 时间码：00:00-00:05
- 景别：全景
- 内容：[画面描述]
- 关键帧路径：`镜头1_关键帧.png`

### 镜头2
- 时间码：00:05-00:08
- 景别：中景
- 内容：[画面描述]
- 关键帧路径：`镜头2_关键帧.png`
```

## 质量检查清单

- [ ] 画面构图符合镜头语言要求（景别、角度）
- [ ] 角色外观与参考图一致
- [ ] 场景元素与场景参考图一致
- [ ] 同一序列的关键帧之间视觉连贯
- [ ] 画面风格与项目风格一致
- [ ] 关键帧适合作为视频生成参考输入
- [ ] 动作分解的箭头清晰指示运动方向（如适用）
