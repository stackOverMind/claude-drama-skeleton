# CLAUDE.md

本文件为 Claude Code (claude.ai/code) 提供项目指引。

## 项目简介

AI 内容创作多项目仓库。每个子项目有统一的目录结构和创作流程。

## 项目命名

- 前缀：`项目-`
- 格式：`项目-{名称}/`
- 示例：`项目-日漫女孩`、`项目-古风剑客`、`项目-产品宣传片`

## 项目目录结构

```
项目-{名称}/
├── 1-脚本/                    # 视频脚本/文案/剧本
├── 2-素材/                    # AI 生成素材（角色、场景、道具等）
├── 3-故事板/                  # 故事板/关键帧定义
├── 4-视频/                    # 最终视频输出
└── 项目说明.md                # 项目基本信息与规则
```

## 文件放置规则

- 所有内容文件必须放在某个 `项目-{名称}/` 目录下
- 根目录仅允许存在：项目文件夹、`临时/`、`CLAUDE.md`、`README.md`、`.claude/`、配置文件
- 不确定归属的文件放入 `临时/`，后续再整理归档到对应项目

## 项目定位（重要）

本仓库是**多项目容器**，根目录本身不是创作项目。

- **项目** = `项目-{名称}/` 子目录，如 `项目-日漫女孩/`
- **仓库根目录** = 仅用于存放多个项目、配置文件和临时文件
- 所有 AI 生成、剧本创作、素材管理等操作，都在具体的 `项目-*/` 目录下进行
- 当用户说「这个项目」时，通常指当前工作所在的某个 `项目-*/` 子目录，而非仓库根目录

## 项目级规则

每个项目目录下都包含 `项目说明.md`，其中记录了该项目特有的规则和配置：

- **项目类型** — 单集视频 / AI 漫剧 / 小说等
- **项目描述** — 项目简介
- **画面风格提示词** — AI 生成素材时的统一视觉风格锚定
- **世界观设定** — 故事世界观、背景设定、时代/地域/规则等
- **备注** — 其他需要记录的信息

当在某个项目内工作时，**优先遵循该项目 `项目说明.md` 中的规则**。全局规则（本文件）与项目规则冲突时，以项目规则为准。
